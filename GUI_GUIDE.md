# 🖥️ GUI 버전 사용 가이드

PyQt5 기반 데스크톱 애플리케이션

## 🚀 실행 방법

### 방법 1: 스크립트 실행 (권장)
```bash
./run_gui.sh
```

### 방법 2: 직접 실행
```bash
uv run python gui_app.py
```

## 📱 GUI 화면 구성

### 1️⃣ 번역 탭
메인 번역 화면

#### 📥 입력 파일
- **파일 선택 버튼**: 번역할 파일 선택
- **지원 형식**: .txt, .docx, .pdf
- 파일 경로가 자동으로 표시됨

#### 📤 출력 파일
- **파일 선택 버튼**: 저장할 파일 선택
- **지원 형식**: .txt, .docx (Word 권장!)
- 입력 파일 선택 시 자동으로 경로 생성 (`파일명_ko.docx`)

#### ⚙️ 번역 옵션
- **문서 유형 선택**:
  - 청구항 (claim) - 기본값
  - 명세서 (specification)
  - 요약서 (abstract)

- **자체 검수 활성화** (체크박스):
  - ✅ 활성화 (권장): 번역 후 자동 검수 및 수정
  - ⬜ 비활성화: 빠른 번역 (검수 생략)

- **Translation Memory에 저장** (체크박스):
  - ✅ 활성화 (권장): 번역 결과를 TM에 저장
  - ⬜ 비활성화: TM 저장 안 함

#### 🚀 번역 시작 버튼
- 클릭하면 번역 시작
- 진행 중에는 버튼 비활성화

#### 📋 진행 상황
- 실시간 로그 표시
- 진행 단계별 메시지 출력
- 번역 결과 미리보기

### 2️⃣ Translation Memory 탭
TM 통계 확인

#### 📊 통계 정보
- 총 항목 수
- 도메인별 분포
- 문서 유형별 분포

#### 🔄 새로고침 버튼
- TM 통계 업데이트

### 3️⃣ 설정 탭
시스템 정보 및 도움말

#### 📋 시스템 정보
- 버전 정보
- 지원 파일 형식
- 번역 엔진 정보

#### 💡 사용 팁
- 권장 사용법
- 최적화 팁

## 🎯 사용 예시

### 기본 사용법
1. **"📝 번역" 탭** 선택
2. **입력 파일 선택**: "파일 선택" 버튼 클릭 → 번역할 Word 파일 선택
3. **출력 파일 확인**: 자동 생성된 경로 확인 또는 변경
4. **문서 유형 선택**: "청구항 (claim)" 선택
5. **옵션 확인**: "자체 검수" 및 "TM 저장" 모두 체크 (권장)
6. **🚀 번역 시작** 버튼 클릭
7. 진행 상황 모니터링
8. 완료 메시지 확인

### 빠른 번역 (검수 생략)
1. 입력/출력 파일 선택
2. **"자체 검수 활성화"** 체크 해제
3. 번역 시작

### TM 활용
1. 동일한 파일을 다시 번역하면 TM에서 자동으로 찾아 즉시 반환
2. **"📊 Translation Memory" 탭**에서 통계 확인

## ⚡ 단축키

- **Ctrl+O**: 입력 파일 선택 (계획)
- **Ctrl+S**: 출력 파일 선택 (계획)
- **Ctrl+T**: 번역 시작 (계획)

## 🎨 UI 특징

### 색상 코드
- 🟢 **초록색 버튼**: 번역 시작
- 🔵 **파란색**: 정보 메시지
- 🟡 **노란색**: 경고
- 🔴 **빨간색**: 오류

### 실시간 피드백
- 진행 바: 번역 중 애니메이션 표시
- 상태바: 현재 작업 상태 표시
- 로그: 상세 진행 내역

## ❓ 문제 해결

### Q: GUI가 실행되지 않아요
```bash
# PyQt5 재설치
uv add --reinstall pyqt5
```

### Q: 파일 선택 창이 안 열려요
- 권한 문제일 수 있습니다
- 터미널에서 실행: `./run_gui.sh`

### Q: 번역 중 멈춘 것 같아요
- 진행 상황 로그 확인
- PDF 파일은 변환 시간이 오래 걸릴 수 있음
- 인터넷 연결 확인 (Claude API 통신)

### Q: 번역 결과가 표시되지 않아요
- 출력 파일 경로 확인
- 파일 저장 권한 확인
- 로그에서 오류 메시지 확인

## 💡 사용 팁

### 1. 실무 최적 설정
```
✅ 입력: Word 파일 (.docx)
✅ 출력: Word 파일 (.docx)
✅ 자체 검수: 활성화
✅ TM 저장: 활성화
✅ 문서 유형: 청구항 (claim)
```

### 2. 배치 처리
- 여러 파일을 순차적으로 번역하려면
- 각 파일마다 입력/출력 선택 후 번역 실행
- TM이 쌓여서 점점 빨라짐

### 3. 품질 우선
- 자체 검수 활성화 (약간 느리지만 품질 향상)
- QA 리포트 확인 (출력파일.qa.txt)

### 4. 속도 우선
- 자체 검수 비활성화
- 텍스트 출력 (.txt)

## 📚 추가 정보

### CLI vs GUI

|------|-----|-----|
| 사용 편의성 | ⭐⭐ | ⭐⭐⭐ |
| 배치 처리 | ⭐⭐⭐ | ⭐⭐ |

### 언제 GUI를 사용하나?
- ✅ 개별 파일 번역
- ✅ 실시간 진행 상황 확인 필요
- ✅ 컴퓨터 작업에 익숙하지 않은 사용자
- ✅ 시각적 인터페이스 선호

### 언제 CLI를 사용하나?
- ✅ 배치 처리 (여러 파일 자동)
- ✅ 스크립트 통합
- ✅ 원격 서버 실행
- ✅ 고급 사용자

## 🎉 GUI 버전의 장점

1. **직관적**: 클릭 몇 번으로 번역 완료
2. **시각적 피드백**: 진행 상황을 실시간으로 확인
3. **오류 처리**: 친절한 오류 메시지
4. **TM 통계**: 한 눈에 확인 가능
5. **초보자 친화적**: 명령어 몰라도 사용 가능
